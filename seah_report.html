프로젝트 개요

본 섹션에서는 프로젝트 추진 배경과 명확한 목표를 설명합니다. 현재 시스템의 한계를 진단하고, 이번 고도화를 통해 달성하고자 하는 정성적, 정량적 목표를 구체적으로 제시합니다.

추진 배경 및 필요성

철강 산업의 복잡한 연속 공정 환경에서 현장의 정확하고 신속한 데이터 처리는 생산성과 안전에 직결됩니다. 현재 사용 중인 PDF 기반 정보 전달 방식과 노후화된 PDA 시스템은 정보 접근성 저하, 데이터 활용의 한계, 비직관적 UX/UI, 유지보수 및 확장성 문제 등 명확한 한계를 가지고 있습니다. 성공적인 스마트 팩토리 전환을 위해서는 현장 작업자 중심의 모바일 앱 구축이 필수적입니다.

프로젝트 목표

정성적 목표

  • 정보 접근성 극대화: 모바일 앱을 통한 실시간 데이터 조회
  • 데이터 기반 의사결정 지원: 직관적 데이터 시각화 대시보드 제공
  • Paperless 환경 구축: 모든 보고 및 확인 절차의 디지털화

정량적 목표 (KPI)

  • 정보 조회 및 업무 처리 시간 50% 단축
  • 데이터 입력 오류율 30% 감소
  • 현장 작업자 직무 만족도 80점 이상 달성

프로젝트 범위

프로젝트의 성공은 명확한 범위 설정에서 시작됩니다. 여기서는 개발에 포함될 핵심 기능(In-Scope)과 이번 단계에서는 제외될 항목(Out-of-Scope)을 명확히 구분하여 프로젝트의 집중도를 높입니다.

In-Scope (개발 포함)

공통/핵심

스마트 로그인, 대시보드, 바코드 스캔, 알림 센터, 디지털 문서 뷰어

물류 관리

입/출고, 재고 관리, 야드 및 위치 관리, 이력 추적

생산 관리

공정 관리 및 보고, 품질 및 중량 확인, 자재 매칭 및 확인

Out-of-Scope (개발 제외)

서버/DB 신규 구축

기존 MES/ERP 시스템과의 API 연동을 전제로 합니다.

관리자용 웹 어플리케이션

현장 작업자용 모바일 앱 개발에 집중합니다.

AI 기반 예측 분석 기능

설비 고장, 수요 예측 등은 향후 고도화 과제로 분류합니다.

구축 방안

성공적인 결과물을 위해 검증된 개발 방법론과 최신 기술 스택을 적용합니다. 애자일 스크럼 방식을 통해 유연하게 변화에 대응하고, 네이티브 앱 기술로 최고의 성능과 안정성을 보장합니다.

개발 방법론: Agile Scrum

2주 단위의 '스프린트'를 반복하여 실제 작동하는 결과물을 지속적으로 제공하고, 현장 사용자의 피드백을 즉시 반영하여 앱의 완성도를 높여갑니다.

일정 및 인력

총 5개월의 프로젝트 기간 동안 각 단계별 활동 계획과 투입될 전문 인력 구성을 시각적으로 보여줍니다. 차트를 통해 전체적인 자원 배분 현황을 파악할 수 있습니다.

5개월 일정 계획

  • 1개월차 (분석 및 기획): 요구사항 분석 및 개발 범위 확정
  • 2개월차 (설계): 화면 설계, UX/UI 디자인, 프로토타입 제작
  • 3개월차 (개발): 핵심 기능 및 API 연동 개발
  • 4개월차 (개발 및 테스트): 전체 기능 개발 완료, 베타 테스트 진행
  • 5개월차 (안정화 및 배포): 최종 QA, 앱 배포, 사용자 교육

투입 인력 계획 (총 20 M/M)

수행 조직
